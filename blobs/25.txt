# SUPER QWEN RPG 2 — RPG 확장 개발 상세 플랜

> 상태: 구현 완료 기준 문서  
> 대상 저장소: `dextune/super-llmario`  
> 릴리스 목표: 브라우저 기반 싱글 플레이 액션 RPG 2.0  
> 문서 기준일: 2026-07-23

## 1. 문서 목적

기존 버전은 레벨, 장비, 능력치와 전리품이 존재하지만 실제 진행은 **오른쪽으로 이동해 적을 처치하고 깃발에 도달하는 직선형 스테이지**에 가까웠습니다. 이 문서는 전투에서 얻은 보상이 마을에서의 선택으로 이어지고, 그 선택이 다음 전투 방식과 성장 방향을 바꾸는 완결된 RPG 루프를 정의합니다.

본 릴리스의 핵심 성공 기준은 다음 한 문장으로 요약합니다.

> 전투 → 전리품 → 비교와 정비 → 성장 선택 → 더 어려운 지역이라는 순환이 한 세션 안에서 반복되어야 한다.

## 2. 기존 구조의 핵심 문제

### 2.1 플레이 흐름

- 여섯 스테이지가 동일한 횡스크롤 진행 구조를 공유했습니다.
- 지역 사이에 정비, 선택, 소비가 없어 스테이지 클리어가 다음 스테이지 로드로만 연결됐습니다.
- 골드는 획득량에 비해 소비처가 부족해 RPG 재화보다 점수처럼 작동했습니다.
- 퀘스트, 거점, 분기 목표, 반복 도전 보상이 없었습니다.

### 2.2 성장과 빌드

- 레벨업 후 기본 능력치 배분은 가능했지만 전투 동작 자체는 거의 바뀌지 않았습니다.
- 주문과 공격이 처음부터 고정되어 직업 정체성과 장기 빌드 선택이 약했습니다.
- 장비 희귀도와 옵션은 존재했지만 판매, 분해, 강화, 재련과 비교 UI가 부족했습니다.

### 2.3 전투 콘텐츠

- 적 행동이 접근 후 접촉 공격 중심이었습니다.
- 원거리, 회복, 방패, 돌진, 소환처럼 역할이 분명한 적 조합이 부족했습니다.
- 최종 보스의 전투 단계와 패턴 전환이 제한적이었습니다.
- 지역별 환경 규칙이 배경색과 적 배치 차이에 머물렀습니다.

### 2.4 지속성과 품질

- 장기 캠페인을 위한 저장·불러오기와 저장 데이터 검증이 필요했습니다.
- 게임오버 상태와 재도전·귀환 동선이 완전히 연결되지 않았습니다.
- 자동 테스트가 시각 렌더링 및 전체 캠페인 흐름까지 충분히 검증하지 못했습니다.

## 3. 제품 목표와 비목표

### 3.1 제품 목표

1. **마을 중심 메타 루프**를 도입합니다.
2. 세 직업이 J/K/L 전투 방식과 스킬 트리에서 명확히 구분되도록 합니다.
3. 골드와 재료에 반복 가능한 소비처를 제공합니다.
4. 여섯 지역마다 목표와 환경 규칙을 다르게 구성합니다.
5. 역할 기반 적, 엘리트 속성, 3단계 최종 보스로 전투 조합을 확장합니다.
6. 자동 저장, 이어하기, 사망 페널티와 재도전을 완결합니다.
7. 핵심 규칙을 DOM과 분리해 Node 환경에서 결정적으로 테스트합니다.

### 3.2 이번 릴리스의 비목표

- 서버 계정, 온라인 멀티플레이, 순위표
- 외부 에셋 파이프라인 또는 대용량 사운드·그래픽 리소스
- 무한 절차 생성 던전
- 거래소나 다른 플레이어와의 아이템 교환
- 모바일 네이티브 앱 패키징

비목표는 확장 가능성을 막지 않되 2.0 캠페인의 완결성과 테스트 가능성을 우선하기 위해 제외합니다.

## 4. 핵심 게임 루프

```text
타이틀
  ↓
직업 선택 또는 이어하기
  ↓
새벽 마을
  ├─ 상점: 장비·물약 구매
  ├─ 대장간: 강화·옵션 재련
  ├─ 스킬 성소: 직업 특성 투자
  ├─ 퀘스트 게시판: 진행 확인·보상 수령
  └─ 장비와 가방: 비교·착용·판매·분해
  ↓
지역 선택
  ↓
필드 전투 + 지역 목표 + 상자 + 엘리트
  ↓
관문 또는 보스
  ↓
별 등급·경험치·골드·재료·장비 보상
  ↓
마을 귀환 및 다음 빌드 선택
```

핵심은 마을이 단순 메뉴가 아니라 **전투 보상을 의사결정으로 변환하는 장소**가 되는 것입니다.

## 5. 시스템 아키텍처

### 5.1 진입점과 UI

- `index.html`은 타이틀, 직업 선택, 마을, 상점, 대장간, 스킬, 퀘스트, 인벤토리, 일시정지, 클리어, 승리, 패배 오버레이를 선언합니다.
- `css/rpg2.css`는 메타 게임 UI, 장비 카드, 반응형 레이아웃을 담당합니다.
- canvas는 필드, 적, 투사체, 지역 환경, HUD 렌더링에 사용합니다.

### 5.2 순수 도메인 계층

전체 규칙은 `rpg2-core.js` 원본을 구성하는 `js/core.parts/*.part`에 저장합니다.

담당 범위:

- 직업·스킬 트리 정의
- 경험치와 레벨업
- 능력치 계산
- 장비 생성, 희귀도, 옵션, 유일 장비
- 착용, 판매, 분해, 강화, 재련
- 상점 재고와 구매
- 퀘스트 진행과 보상
- 스테이지 보상과 별 등급
- 저장 데이터 정규화와 직렬화

이 계층은 DOM, canvas, WebAudio, localStorage에 의존하지 않습니다. 따라서 동일 입력과 난수 시드를 사용하면 동일 결과를 재현할 수 있습니다.

### 5.3 게임 런타임 계층

`rpg2-game.js` 원본을 구성하는 `js/game.parts/*.part`는 다음을 담당합니다.

- 상태 머신과 화면 전환
- 입력, 이동, 점프, 공격과 기술
- 필드 생성과 카메라
- 적 AI, 투사체, 피격과 사망
- 지역 기믹과 목표 판정
- 드롭, 상자, 관문
- HUD와 canvas 렌더링
- localStorage 자동 저장
- 테스트용 런타임 스냅샷 API

### 5.4 소스 조립과 무결성

대형 소스는 GitHub API와 정적 호스팅 환경에서 안정적으로 다룰 수 있도록 읽을 수 있는 UTF-8 조각으로 저장합니다.

- `js/source-manifest.json`: 조각 순서, 원본 바이트 길이, SHA-256
- `js/bootstrap.js`: 조각 로드 → 연결 → 길이·SHA-256 검증 → 코어와 런타임 순서대로 평가
- `scripts/assemble-sources.js`: Node에서 동일 규칙으로 소스를 조립하고 문법·해시를 검사

브라우저는 정적 웹 서버에서 실행해야 하며, 파일이 누락되거나 순서가 변하면 부팅을 중단하고 오류 상태를 표시합니다.

### 5.5 상태 머신

```text
TITLE → CLASS → HUB
TITLE → HUB (이어하기)
HUB → SHOP | FORGE | SKILLS | QUESTS | INVENTORY | FIELD
FIELD → PAUSED | INVENTORY | CLEAR | WIN | OVER
CLEAR | WIN | OVER → HUB
```

필드 외 메타 화면에서는 전투 틱을 멈춥니다. 인벤토리를 필드에서 열면 닫을 때 동일 필드로 복귀하고, 마을에서 열면 마을로 복귀합니다.

## 6. 직업과 빌드 설계

### 6.1 선봉대

- 정체성: 높은 생명력, 근접 압박, 방어 태세
- J: 3연속 베기
- K: 대지 강타 — 넓은 충격파와 방패 파괴
- L: 철벽 태세 — 짧은 시간 피해 감소
- 성장 축: 공격력, 방어, 강타 범위, 처형, 처치 시 강타 초기화

### 6.2 비전술사

- 정체성: 원거리 주문, 화염 지속 피해, 냉기 군중 제어
- J: 비전탄
- K: 화염구 — 분열 투사체와 화상
- L: 서리 폭발 — 광역 피해와 빙결
- 성장 축: 스킬 피해, 마나 순환, 투사체 수, 동결, 연쇄 연소, 마나 비용 감소

### 6.3 척후병

- 정체성: 기동성, 치명타, 다중 사격
- J: 빠른 원거리 기본 공격
- K: 다중 사격
- L: 그림자 질주 — 이동과 무적, 주변 타격
- 성장 축: 치명타, 이동·회피, 화살 수, 질주 강화, 엘리트·보스 피해, 무료 사격

### 6.4 성장 규칙

- 레벨업마다 스킬 포인트 1과 능력치 포인트 3을 지급합니다.
- 능력치 포인트는 완력, 활력, 집중, 기민에 배분합니다.
- 각 직업은 6개 스킬 노드를 가지며 일부 노드는 선행 랭크를 요구합니다.
- 최종 노드는 비용 2의 빌드 전환형 효과로 설계합니다.

## 7. 아이템과 경제

### 7.1 장비 구조

- 슬롯: 무기, 갑옷, 투구, 장화, 반지, 목걸이
- 희귀도: 일반, 마법, 희귀, 영웅, 유일
- 일반 장비는 기본 능력치를, 상위 희귀도는 추가 옵션을 가집니다.
- 유일 장비는 숫자뿐 아니라 전투 동작을 바꾸는 고유 효과를 가집니다.

예시 고유 효과:

- 대지 강타의 추가 충격파
- 스테이지당 한 번 치명상 방지
- 빙결 대상 추가 피해
- 질주 경로 번개 흔적
- 치명타 처치 시 물약 획득 가능성
- 높은 마나에서 주문 피해 증가

### 7.2 재화

- 골드: 상점 구매, 상점 갱신, 강화, 재련
- 재료: 강화와 재련
- 장비 판매: 골드 획득
- 장비 분해: 희귀도와 강화 수치에 따라 재료 획득

### 7.3 상점

- 캐릭터 레벨을 기준으로 결정적 재고를 생성합니다.
- 장비 6개와 체력·마나 물약을 제공합니다.
- 장비는 한 재고 주기에서 한 번만 구매할 수 있습니다.
- 상품 교체 비용은 교체 횟수에 따라 증가합니다.

### 7.4 대장간

- 강화는 최대 +7입니다.
- 강화할수록 골드와 재료 비용이 증가합니다.
- 강화는 장비의 양수 능력치를 비율로 높입니다.
- 마법 이상, 유일 미만 장비는 마지막 옵션을 재련할 수 있습니다.
- 재련은 기존 옵션을 제거한 뒤 같은 슬롯에서 유효한 다른 옵션을 생성합니다.

## 8. 퀘스트

모든 퀘스트는 자동 추적하며 마을 게시판에서 수령합니다.

| 퀘스트 | 조건 | 주요 보상 |
|---|---|---|
| 길 위의 위협 | 적 20마리 처치 | 골드·재료 |
| 잊힌 보물 | 상자 5개 개방 | 골드·재료 |
| 강자의 증명 | 엘리트 4마리 처치 | 골드·스킬 포인트 |
| 대장장이의 부탁 | 장비 3회 강화 | 골드·재료 |
| 귀환 없는 승리 | 무사망 클리어 2회 | 골드·스킬 포인트 |
| 왕의 몰락 | 최종 보스 처치 | 골드·재료·유일 장비 |

보상 수령은 중복 실행되지 않으며, 유일 장비 보상 시 가방 공간을 먼저 검증합니다.

## 9. 지역별 콘텐츠

| 지역 | 핵심 목표 | 환경·전투 규칙 |
|---|---|---|
| 새벽 초원 | 적 8마리 처치 후 관문 | 기본 전투와 이동 학습 |
| 황혼 벌판 | 엘리트 처치 후 탈출 | 엘리트 속성 첫 노출 |
| 메아리 동굴 | 상자에서 열쇠 획득 | 잠긴 관문과 탐색 동기 |
| 백야 설원 | 관문까지 생존 | 동상 게이지, 화톳불 안전지대 |
| 용암 균열 | 균열 핵 3개 파괴 | 주기적 용암 파동, 높은 지대 회피 |
| 마왕성 | 쿠파 대왕 3단계 처치 | 패턴 전환, 추가 적, 광역 공격 |

### 9.1 별 등급

- 기본 클리어: 1성
- 지역 목표 달성: +1성
- 기준 시간 이내 및 무사망: +1성

별 수에 따라 골드, 경험치, 재료와 보상 장비의 품질이 상승합니다. 최고 기록은 별 수를 우선하고 같은 별 수에서는 더 빠른 시간을 저장합니다.

## 10. 적과 전투 역할

### 10.1 일반 역할

- 근접형: 단순 추격과 접촉 공격
- 방패형: 정면 피해 감소, 강타·콤보 마무리로 대응
- 비행형: 공중에서 플레이어를 추적
- 원거리형: 적정 거리를 유지하며 투사체 발사
- 회복형: 주변 아군 우선 회복, 대상이 없으면 공격
- 돌진형: 경고선 후 고속 돌진
- 소환형: 하수인 수를 제한하며 주기적으로 소환

### 10.2 엘리트 속성

- 광폭: 빠른 이동과 높은 공격력
- 요새: 높은 생명력
- 흡혈: 가한 피해 기반 회복
- 냉기 오라: 주변 플레이어 둔화
- 폭발: 사망 시 범위 피해
- 점멸: 주기적 순간이동

엘리트는 색상, 테두리와 이름표로 식별되며 더 높은 장비 드롭 확률을 가집니다.

### 10.3 최종 보스 3단계

1. 1단계: 전방 화염 투사체와 기본 추격
2. 2단계: 추가 적 소환, 공격 간격 단축
3. 3단계: 더 많은 투사체, 충격파, 하수인 소환 강화

체력 비율에 따라 단계가 전환되며 HUD와 알림으로 전환을 명시합니다.

## 11. 저장과 실패 복구

### 11.1 저장 스키마

- 저장 버전: 2
- 저장 항목: 직업, 레벨, 경험치, 재화, 스킬·능력치, 장비·가방, 물약, 지역 해금·기록, 퀘스트, 통계, 설정
- 저장 시 모든 아이템과 프로필 필드를 정규화합니다.
- 버전이 맞지 않거나 필수 직업 데이터가 없는 저장은 거부합니다.

### 11.2 저장 시점

- 새 캐릭터 생성
- 마을 진입
- 상점·대장간·스킬·퀘스트·인벤토리 변경
- 스테이지 종료
- 필드 진행 중 25초 간격 자동 저장
- 타이틀 복귀

### 11.3 사망 처리

- 사망 시 현재 골드의 10%를 잃습니다.
- 재도전 또는 마을 귀환을 선택할 수 있습니다.
- 불사조 유일 갑옷은 스테이지당 한 번 사망을 막습니다.
- 낙하와 시간 초과도 같은 실패 플로우를 사용합니다.

## 12. UI와 접근성

- 키보드: A/D 또는 방향키, Space, J/K/L, Q/E, I, P/Escape
- 마우스: 좌클릭 기본 공격, 우클릭 주력기
- 모바일: 이동·점프·공격·기술·물약 터치 버튼
- 장비 카드에 희귀도 색상, 슬롯, 전투력, 능력치를 표시합니다.
- 선택 장비는 현재 장비와의 전투력 차이를 표시합니다.
- 반응형 레이아웃으로 작은 화면에서 목록을 단일 열 또는 축소 그리드로 전환합니다.
- 부팅 실패 시 정적 서버 실행 여부를 확인할 수 있는 사용자 메시지를 표시합니다.

## 13. 밸런스 목표

### 13.1 캠페인 진행

- 1회 캠페인은 숙련도에 따라 약 35~60분을 목표로 합니다.
- 각 지역은 일반 플레이 4~8분, 최종 지역은 8~12분을 목표로 합니다.
- 첫 지역 종료 시 최소 1개 신규 장비와 1회 이상의 소비 결정을 제공해야 합니다.

### 13.2 경제

- 일반 스테이지 1회 보상으로 현재 레벨 장비 한 개 또는 1~2회의 저단 강화 중 하나를 선택할 수 있어야 합니다.
- 상점 갱신은 반복 가능하지만 연속 갱신 비용이 빠르게 증가해야 합니다.
- 분해 재료는 강화 비용을 완전히 대체하지 않고 파밍과 소비 사이의 완충 역할을 합니다.

### 13.3 전투

- 동레벨 일반 적은 기본 공격 약 2~5회 내 처치를 목표로 합니다.
- 엘리트는 일반 적보다 약 1.5~2.5배 긴 교전을 목표로 합니다.
- 보스는 패턴을 보지 않고 접촉 공격만 반복해서는 안정적으로 이길 수 없어야 합니다.
- 방어형 적은 정면 공격만 반복하는 플레이를 명확히 손해 보게 해야 합니다.

## 14. 테스트 전략

### 14.1 소스 무결성

- manifest의 조각 순서대로 연결합니다.
- 연결 결과의 바이트 길이와 SHA-256을 검사합니다.
- 조립한 코어와 런타임의 JavaScript 문법을 컴파일합니다.

### 14.2 도메인 단위 테스트

- 세 직업 프로필 생성과 시작 장비
- 경험치·레벨업·포인트 지급
- 스킬 선행 조건과 최대 랭크
- 결정적 아이템 생성과 희귀도·슬롯 유효성
- 착용·해제·판매·분해
- 상점 재고, 단일 구매, 물약 구매와 갱신
- 강화 비용·최대 랭크·재련
- 퀘스트 진행·중복 수령 방지
- 스테이지 별 등급, 보상, 다음 지역 해금
- 저장 직렬화·복원과 잘못된 저장 거부

### 14.3 헤드리스 런타임 스모크 테스트

DOM, canvas 2D, localStorage, Audio, requestAnimationFrame을 스텁한 VM에서 다음을 검증합니다.

1. 코어와 런타임이 예외 없이 부팅됩니다.
2. 새 직업 생성 후 마을 상태에 진입합니다.
3. 각 지역을 로드하고 이동·공격·두 기술·물약·렌더를 호출합니다.
4. 관문 목표를 충족시켜 여섯 지역 전체 플로우를 완료합니다.
5. 최종 지역 후 승리 상태와 캠페인 완료 플래그를 검증합니다.
6. 저장 데이터가 localStorage에 생성되고 다시 불러와집니다.
7. canvas 그리기 호출과 RAF 루프가 지속됩니다.

### 14.4 CI

GitHub Actions는 Node 20에서 `npm test`를 실행합니다. 외부 패키지 설치가 필요하지 않으므로 네트워크 상태와 무관하게 동일 검증이 가능합니다.
